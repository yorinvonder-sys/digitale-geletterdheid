# Werkstromen (Workstreams)

Elke taak hoort bij één werkstroom. Label altijd de actieve werkstroom in het statusblok.

---

## Actieve werkstromen

### 1. Product
Code en features bouwen voor het platform zelf.
- Missies, dashboard, onboarding, games, assessment
- UI/UX verbeteringen, bugfixes, performance
- Template engine, mission registry

### 2. Compliance
Privacy, security, en juridische compliance in de codebase.
- Compliance Hub pagina
- AI Act code-taken (Art. 9, 12, 14)
- RLS policies, audit logging, AI-transparantie
- DPIA/DPA updates als de code verandert

### 3. Go-to-Market
Alles wat nodig is om scholen binnen te halen.
- Homepage conversie-optimalisatie
- SEO landing pages
- Outreach templates en content
- Pilot-aanmeldformulier

### 4. Infra
Ontwikkeltools, CI/CD, projectstructuur.
- `.claude/` bestanden (task tracking, handoff)
- Build pipeline, Vercel config
- Database migraties, edge functions setup

### 5. Pilot
Taken specifiek voor het draaien van de pilot.
- Docent-onboarding flow
- Feedback verzameling
- KPI tracking
- Wekelijkse iteratie op basis van feedback

---

## Regels

- **Eén werkstroom tegelijk actief** in het statusblok
- **Scopewissel expliciet benoemen:** "We wisselen nu van Product naar Compliance"
- **Niet springen** tussen werkstromen zonder het te benoemen
