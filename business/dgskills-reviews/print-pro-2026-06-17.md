## 🎨 Design review

### ✅ Goed

- **Tokens correct:** ToolGuide.tsx gebruikt uitsluitend `duck-ink`, `duck-acid`, `duck-bg`, `duck-bgLight`, `duck-gray` — alle gedefinieerd in `tailwind.shared.js:8-15`. Geen hex-literals in JSX.
- **Focus-visible:** `focus-visible:ring-2 focus-visible:ring-duck-acid` op alle interactieve elementen (ToolGuide.tsx:234, 342, 374). Voldoet aan WCAG 2.1 AA 2.4.7.
- **Knop-duidelijkheid:** Checklist-items, quiz-opties en CTA's hebben labels. DuckMark + Check icon enkel als versiering, geen standalone icon-knoppen.
- **Responsive:** `max-w-md mx-auto w-full` — geen vaste px-widths. Correct.
- **Framer Motion:** Geen `motion.*` of `AnimatePresence` in ToolGuide.tsx — intentioneel absent.

### ⚠️ Aandachtspunten

- **Badge-kleuren identiek (alle drie `#202023`):**
  Drie badges zijn visueel ononderscheidbaar bij completion. Niet werkend als motivatie-trigger.
  `print-pro.ts:109-123`
  ```ts
  // Voor:
  { minScore: 55, emoji: '🏆', title: 'Print Expert', color: '#202023' },
  { minScore: 40, emoji: '🖨️', title: 'Print Pro',    color: '#202023' },
  { minScore: 0,  emoji: '🌱', title: 'Aan de slag',  color: '#202023' },

  // Na:
  { minScore: 55, emoji: '🏆', title: 'Print Expert', color: '#D97848' },  // lab-coral / warm goud
  { minScore: 40, emoji: '🖨️', title: 'Print Pro',    color: '#445865' },  // lab-muted / blauwgrijs
  { minScore: 0,  emoji: '🌱', title: 'Aan de slag',  color: '#7B8794' },  // neutraal grijs
  ```

- **Stap 2 instructie > 60w (leerjaar 1 grens):**
  Drie platforms (iPad / Chromebook / Windows) in één aaneengesloten zin. ~65w, grens 60w voor jr1.
  `print-pro.ts:47`
  ```ts
  // Voor (65w):
  'Open de **print-app** van je school ... Log in met je **schoolaccount** als de app daarom vraagt.'

  // Na (~42w — platform-keuzelijst):
  'Open de print-app of de ingebouwde printfunctie van je apparaat:\n\n- **iPad:** Deel > Afdrukken → kies de schoolprinter\n- **Chromebook/Windows:** Ctrl+P → kies de printer\n\nLog in met je schoolaccount als dat gevraagd wordt.'
  ```

- **Stap 1 self-declare checklistItem zonder teacherCheck:**
  "Ik weet welk printsysteem mijn school gebruikt" is buiten de app verifieerbaar en ongevalideerd.
  `print-pro.ts:27`
  ```ts
  // Voeg toe aan stap 1 (na checklistItems):
  teacherCheck: 'Laat je docent zien welk printsysteem jouw school gebruikt.'
  ```

- **Visuele QA — niet geverifieerd (geen screenshot):**
  Geen schermafbeeldingen beschikbaar in `screenshots/assignments/print-pro/`. Multi-viewport QA (375px / iPad / desktop) en checklist-gate-gedrag zijn statisch niet verifieerbaar.

## 📚 Didactiek review

### ✅ Goed

- **missionGoal aanwezig:** `missionGoals.ts:41` — `primaryGoal: 'Ik print en lever een schoolbestand correct in met de juiste printerinstellingen.'` Concreet en actiegericht. Engine haalt dit op via `getMissionGoal('print-pro')` als fallback.
- **Leeftijdspassend vocabulary:** "App Library", "Ctrl+P", "FollowMe" worden contextueel uitgelegd. Geen onverklaard jargon.
- **Curriculum-plek logisch:** Leerjaar 1, week 1 — afsluiting van "Digitale Basisvaardigheden" na magister-master, word-wizard, slide-specialist. Didactisch logische volgorde.
- **Wellbeing/inclusiviteit:** Geen gevoelige onderwerpen. VSO-mapping aanwezig (`18A`). Inclusief taalgebruik.
- **SLO-codes geldig:** `21A` en `18A` zijn geldige codes in de mapping.
- **Bloom-activatie in 3 van 4 stappen:** Stap 1 (recall), Stap 3 (toepassen/kiezen), Stap 4 (oorzaak-gevolg redeneren) hebben elk een verificationQuestion.

### ⚠️ Aandachtspunten

- **SLO 23A impliciet aanwezig, niet geclaimd:**
  Stap 4 introduceert expliciet FollowMe-authenticatie en printqueue-privacy (`print-pro.ts:83`). Dit is 23A-materiaal (digitale veiligheid + privacy). De SLO-claim dekt de inhoud niet volledig.
  `slo-kerndoelen-mapping.ts:32`
  ```ts
  // Voor:
  { id: 'print-pro', ..., sloKerndoelen: ['21A'], sloVsoKerndoelen: ['18A'] }

  // Na:
  { id: 'print-pro', ..., sloKerndoelen: ['21A', '23A'], sloVsoKerndoelen: ['18A', '20A'] }
  ```

- **Bloom-balans: stap 2 zonder cognitieve activatie:**
  Meest complexe stap (multi-platform login-flow) mist verificationQuestion. Nul cognitieve activatie op het moeilijkste moment.
  `print-pro.ts:43-54`
  ```ts
  // Voeg toe aan stap 2 (na checklistItems):
  verificationQuestion: {
    question: 'Hoe open je het printvenster op een Chromebook of Windows-laptop?',
    options: [
      'Via App Library > Afdrukken',
      'Ctrl+P in het document',
      'Via Instellingen > Printers > Afdrukken',
      'Door de print-app te installeren',
    ],
    correctIndex: 1,
    explanation: 'Ctrl+P opent het printvenster direct in bijna elk programma — op Chromebook én Windows. Snel en handig!',
  }
  ```

- **Missie-uitvoerbaarheid thuis:**
  Stappen 2–4 vereisen een fysieke schoolprinter. Een thuisleerling kan deze stappen niet voltooien. De missie is feitelijk een veldopdracht.
  Voorstel: voeg instructienoot toe in `introDescription` of als tip bij stap 1:
  > "Tip: Voer deze missie uit op school, in de buurt van een printer."

- **SLO 21A procedureel zonder systeembegrip:**
  De missie dekt de handelingen van 21A, maar legt niet uit hoe een printsysteem architectureel werkt (client-server, wachtrij-authenticatie). Verdedigbaar als startniveau J1, maar de claim is aan de onderkant van 21A.

### ❌ Blokkers

- **BLOCKING-DID1 — Geen Bloom-geformuleerde learningObjectives:**
  `takeaways` (`print-pro.ts:125-131`) zijn beschrijvende zinnen ("Je weet hoe...", "Je kunt..."), geen toetsbare leer-doelen met actiewerkwoord. Dit voldoet niet aan de didactische eis voor Bloom-geformuleerde doelen. Het type `ToolGuideConfig` heeft geen `learningObjectives`-veld — dit is een ESCALATIE: vereist config-type-uitbreiding en engine-aanpassing (niet auto-fixbaar via config alleen).

  **ESCALATIE (niet auto-fixbaar):** `ToolGuide.tsx` en `ToolGuideConfig`-type hebben geen `learningObjectives`-veld. De missie-content is beschikbaar als takeaways, maar het ontbreekt aan een Bloom-geformuleerde weergave voor docenten. Opties: (a) takeaways herschrijven als actiewerkwoord-zinnen en daarmee de inhoud upgraden, of (b) engine uitbreiden met learningObjectives-veld.

  Interim fix zonder engine-aanpassing — herschrijf `takeaways` met actiewerkwoorden:
  ```ts
  // print-pro.ts:125-131
  // Voor:
  takeaways: [
    'Je weet hoe je het printsysteem van je school herkent en instelt',
    'Je kunt je apparaat verbinden met de schoolprinter',
    ...
  ]

  // Na (Bloom-actiewerkwoord):
  takeaways: [
    'Je herkent het printsysteem van je school en weet waar je die informatie vindt',
    'Je verbindt je apparaat (iPad, Chromebook of laptop) met de schoolprinter via de juiste app',
    'Je past printinstellingen bewust aan (kleur/zwart-wit, dubbelzijdig, paginabereik) op basis van het document',
    'Je legt uit waarom direct ophalen van een printopdracht bijdraagt aan privacy en betrouwbaarheid',
  ]
  ```
  (Actiewerkwoorden: herkent, verbindt, past aan, legt uit — Bloom 1-2 passend bij J1)
